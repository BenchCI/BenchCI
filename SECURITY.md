# Security Policy

BenchCI takes security seriously.

This document describes how to report security vulnerabilities and how they are handled.

---

## Reporting Vulnerabilities

If you discover a security vulnerability, please report it privately.

📧 [tech@benchci.dev](mailto:tech@benchci.dev)

Include:

* description of the vulnerability
* steps to reproduce
* potential impact
* affected components

---

## Responsible Disclosure

Please do not publicly disclose vulnerabilities before they are addressed.

BenchCI will investigate all reports and work to resolve issues promptly.

---

## Scope

Security considerations include:

* licensing and authentication
* remote agent execution
* CI integration
* artifact handling
* communication transports

---

## Response Process

BenchCI follows this process:

1. Acknowledge the report
2. Investigate the issue
3. Develop a fix
4. Release an update
5. Notify affected users if necessary

---

## Security Updates

Security fixes are distributed through updated BenchCI releases.

Users are encouraged to keep their installation up to date.
