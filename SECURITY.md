# Security Policy

BenchCI takes security seriously.

This document describes how to report security vulnerabilities.

---

# Reporting Vulnerabilities

If you discover a vulnerability in BenchCI, please report it privately.

Contact:

tech@benchci.dev

Please include the following information:

- description of the vulnerability
- reproduction steps
- potential impact
- affected components

---

# Responsible Disclosure

Please do not publicly disclose vulnerabilities before they are addressed.

BenchCI will investigate all reports and work to resolve issues promptly.

---

# Security Scope

Security considerations include:

- licensing and authentication
- remote agent execution
- CI integration
- artifact handling
- communication transports

---

# Response Process

1. Acknowledge vulnerability report
2. Investigate issue
3. Develop fix
4. Release updated version
5. Notify affected users if necessary

---

# Security Updates

Security fixes will be released through updated BenchCI distributions.

Users are encouraged to update BenchCI regularly.
